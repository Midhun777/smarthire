ABSTRACT

The Job AI platform is an intelligent recruitment system designed to streamline the hiring process through automated candidate-job matching. The system leverages artificial intelligence to parse resumes, extract relevant skills and experience, and compute compatibility scores between job seekers and available positions. This platform serves three primary user roles: Administrators who manage platform operations, Job Providers who post vacancies, and Job Seekers who apply for positions. The system architecture is built on a relational database model with well-defined entities and relationships, ensuring data integrity and efficient query processing.

1. DATABASE TABLE DEFINITIONS

Description: Stores comprehensive information about all platform users including authentication credentials and role-based access control.

TABLE NAME: tbl_users

Sl no Field Type Description
1 user_id INT Primary Key - Unique user identifier
2 full_name VARCHAR(100) Complete name of the user
3 email_address VARCHAR(100) Unique email for authentication
4 password_hash VARCHAR(255) Encrypted password credential
5 user_role VARCHAR(20) Role type: admin, job_seeker, job_provider
6 created_at DATETIME Account creation timestamp

Description: Contains detailed professional profiles for job seekers including skills and experience extracted from resumes.

TABLE NAME: tbl_seeker_profiles

Sl no Field Type Description
1 profile_id INT Primary Key - Unique profile identifier
2 user_id INT Foreign Key - References tbl_users(user_id)
3 skills_list TEXT Comma-separated technical and soft skills
4 experience_years INT Total years of professional experience
5 resume_text TEXT Parsed resume content for AI analysis
6 is_complete BOOLEAN Profile completion status flag

Description: Stores job vacancy postings with requirements and categorization for efficient matching.

TABLE NAME: tbl_job_listings

Sl no Field Type Description
1 job_id INT Primary Key - Unique job identifier
2 job_title VARCHAR(150) Position title or designation
3 company_name VARCHAR(100) Hiring organization name
4 job_description TEXT Detailed role responsibilities
5 requirements TEXT Required skills and qualifications
6 category_id INT Foreign Key - References tbl_categories(category_id)
7 posted_by INT Foreign Key - References tbl_users(user_id)

Description: Tracks job applications submitted by seekers with status management.

TABLE NAME: tbl_applications

Sl no Field Type Description
1 application_id INT Primary Key - Unique application identifier
2 job_id INT Foreign Key - References tbl_job_listings(job_id)
3 user_id INT Foreign Key - References tbl_users(user_id)
4 application_status VARCHAR(20) Status: Pending, Reviewed, Shortlisted, Rejected
5 applied_date DATETIME Application submission timestamp
6 match_score DECIMAL(5,2) AI-computed compatibility percentage

Description: Defines job categories for classification and filtering purposes.

TABLE NAME: tbl_categories

Sl no Field Type Description
1 category_id INT Primary Key - Unique category identifier
2 category_name VARCHAR(50) Category label (e.g., IT, Healthcare)
3 description VARCHAR(200) Category description

4. CONCLUSION

The Job AI platform provides a comprehensive solution for modern recruitment challenges through intelligent automation. The database architecture ensures normalized data storage with proper referential integrity through foreign key relationships. The Chen ER model clearly illustrates the central role of the User entity and its interactions with dependent entities, all connected through the core Job_Listing entity. The hierarchical data flow diagrams demonstrate the logical flow of information across different user modules, ensuring secure authentication and role-based access control. This system architecture supports scalability and maintainability while providing an efficient framework for AI-powered job matching.
